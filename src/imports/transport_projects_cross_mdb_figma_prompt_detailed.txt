Design a professional web dashboard for a transport project intelligence database covering ADB, AIIB, and World Bank projects.

Primary reference files:
- transport_projects_cross_mdb_figma_profile.txt
- transport_projects_cross_mdb_figma_sample_86.csv

Goal:
Create a clean, modern, executive-friendly dashboard that showcases what the data can do: locations, amounts, recipients, dates, sectors, and project profiles. This is not a data engineering or QA dashboard. It should feel like a transport finance intelligence explorer.

Tone:
Professional, analytical, map-forward, portfolio/profile oriented. Avoid generic startup SaaS visuals. Use strong information hierarchy, restrained color, clear typography, and a polished institutional feel.

Primary user tasks:
1. Understand the overall transport project portfolio
2. Compare ADB, AIIB, and World Bank
3. Explore recipient countries and regions
4. See project distribution over time
5. Filter by transport mode, geography, status, and financing type
6. Open a project detail view with traceability back to the source record

Main dataset in production:
- transport_projects_cross_mdb_shareable_v3.csv

Important data facts:
- 6,372 transport projects
- Sources:
  - World Bank: 4,617
  - ADB: 1,657
  - AIIB: 98
- Mapped projects with coordinates: 2,018
- Geo statuses:
  - hold: 4,313
  - already_geocoded: 1,408
  - matched: 609
  - low_precision: 40
  - review: 1
  - zero_results: 1

Visual guidance:
- Use a major map panel as a signature element
- Use horizontal bars for top countries and sectors
- Use stacked bars for status and source composition
- Use a timeline chart for approval years
- Use a dense but polished explorer table
- Use a right-side drawer for project details
- Use small visual indicators for mapped vs unmapped and low-precision locations

Important UX rules:
- Render blanks as Unknown
- Mark low-precision mapped rows visually
- Allow a Mapped only toggle
- Keep unmapped projects visible in tables and totals, but not on the point map
- Use standardized country values where available
- Preserve original labels and source traceability in the project detail view

Design outcome:
Produce a desktop-first responsive dashboard design system and page layouts suitable for quick implementation in Streamlit or a lightweight web app.
